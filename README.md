## ✨ Características

### 🎯 Sistema de 10 Etapas

Gestiona proyectos direct response con las 10 etapas del proceso:

1. **Investigación** (20%) - Análisis de mercado
2. **Brief** (20%) - Documentación estratégica
3. **Producto** (10%) - Desarrollo de producto
4. **Landing Page** (10%) - Página de ventas
5. **Integraciones** (5%) - Conexiones técnicas
6. **Anuncios** (15%) - Creación de ads
7. **Upsell** (5%) - Ofertas ascendentes
8. **Optimización** (5%) - Mejoras y ajustes
9. **Escala** (5%) - Escalamiento de campañas
10. **Branding/Orgánico** (5%) - Construcción de marca

### 📝 Documentos de Contexto

6 secciones para documentación del proyecto:

- 👤 **Avatar del Cliente** - Perfil ideal
- 🔍 **Investigación de Mercado** - Datos del mercado
- ⚔️ **Competencia** - Análisis competitivo
- 📦 **Producto/Servicio** - Detalles del producto
- 💎 **Oferta Irresistible** - Construcción de oferta
- ✍️ **Headline** - Títulos principales

### 📁 Gestión de Archivos

Sistema híbrido de archivos:

- ✅ **Subida local** - Guardar en vault de Obsidian
- ✅ **Links externos** - Conectar con recursos en la nube
- ✅ **Drag & Drop** - Interfaz intuitiva
- ✅ **Por etapa** - Organizar archivos por cada etapa

### 📊 Contador de Días Porcentual

- 📈 Cálculo automático de días por etapa
- 📊 Porcentaje de progreso visual
- ⚠️ Alertas de retraso
- 🎯 Seguimiento de plazos

### 🗓️ Calendario Integrado

- 📅 3 vistas: mes, semana, día
- 📌 Creación de eventos
- 🔁 Eventos recurrentes
- 🎯 Integración con timeboxing

### 📋 Rutina Diaria

4 secciones de productividad:

**Morning Routine:**
- 🙏 Oración
- 📚 Inglés (reading, listening, vocabulary)
- 🎧 Listening skill

**Study Section:**
- ✍️ Copywriting
- 🎥 VSL (Video Sales Letter)
- 📢 TSL (Text Sales Letter)
- 📖 Libros
- 🎙️ Podcasts

**Timeboxing:**
- ⏱️ Bloques de tiempo
- 🔁 Recurrencia configurable
- 🎯 Vinculación con proyectos

**Todo Lists:**
- 📝 Today
- 📆 This Week
- 🗃️ Later
- 🎯 Prioridades

### 🎨 Liquid Glass UI

- 💜 Branding púrpura (#667eea → #764ba2)
- 🪟 Glassmorphism moderno
- 📱 Responsive design
- 🌈 Gradientes suaves

## 🚀 Instalación

### Opción 1: Desde Community Plugins (Próximamente)

1. Abre Settings → Community Plugins
2. Busca "Direct Response Project Manager"
3. Click en "Install"
4. Habilita el plugin

### Opción 2: BRAT (Beta Tester's Autoupdate)

1. Instala el plugin [BRAT](https://github.com/TfTHacker/obsidian42-brat)
2. Abre Command Palette (Ctrl/Cmd + P)
3. Ejecuta "BRAT: Add a beta plugin for testing"
4. Ingresa: `https://github.com/antigravity/obsidian-direct-response-pm`
5. Habilita el plugin

### Opción 3: Manual

1. Descarga la última release desde [GitHub Releases](https://github.com/antigravity/obsidian-direct-response-pm/releases)
2. Crea carpeta `.obsidian/plugins/direct-response-project-manager/` en tu vault
3. Extrae los archivos descargados en esa carpeta
4. Abre Settings → Community Plugins → Enable "Direct Response Project Manager"

## 📚 Guía Rápida

### 1. Crear Primer Proyecto

1. Abre Command Palette (Ctrl/Cmd + P)
2. Ejecuta "Open Project Dashboard"
3. Click en "➕ Nuevo Proyecto"
4. Ingresa nombre y descripción
5. Configura duración (por defecto: 10 días)

### 2. Configurar Documentos de Contexto

1. En el dashboard, click en "📄 Contexto"
2. Completa cada sección:
   - Avatar del Cliente
   - Investigación de Mercado
   - Competencia
   - Producto/Servicio
   - Oferta Irresistible
   - Headline
3. Guarda cambios

### 3. Gestionar Etapas

1. En el dashboard, verás las 10 etapas
2. Click en una etapa para expandir
3. Agrega tareas con sus pilares:
   - Copywriting
   - Traffic
   - Design
   - Editing
   - Integrations
   - Strategist
4. Marca tareas como completadas
5. Observa el contador de días porcentual

### 4. Subir Archivos

1. En una etapa, click en "📁 Archivos"
2. Elige tipo: "Local" o "Link"
3. Arrastra archivos o pega URL
4. Archivos se organizan por etapa

### 5. Usar el Calendario

1. Abre Command Palette → "Open Calendar"
2. Navega entre vistas: Mes/Semana/Día
3. Crea eventos:
   - Título
   - Tipo (stage, routine, deadline)
   - Fecha y hora
   - Recurrencia (opcional)
4. Drag & drop para reorganizar

### 6. Rutina Diaria

1. Abre Command Palette → "Open Daily Routine"
2. Completa 4 secciones:

**Morning:**
- Oración del día
- Inglés: reading, listening, vocabulary
- Listening skill

**Study:**
- Estudio de copy
- Análisis de VSL
- Estudio de TSL
- Libros y podcasts

**Timeboxing:**
- Crea bloques de tiempo
- Configura recurrencia
- Vincula con proyectos

**Todos:**
- Agrega tareas a hoy/esta semana/más tarde
- Marca prioridades
- Completa tareas

## 🛠️ Arquitectura Técnica

### Stack Tecnológico

- **TypeScript** 5.9+ - Type safety estricto
- **Obsidian API** v1.5+ - Extensión nativa
- **Zustand** - State management con persistencia
- **Rollup** - Bundling optimizado
- **CSS Modules** - Estilos modulares

### Estructura del Proyecto

```
obsidian-project-manager/
├── manifest.json                 # Metadatos del plugin
├── main.ts                      # Punto de entrada
├── package.json                 # Dependencias
├── tsconfig.json                # Configuración TypeScript
├── rollup.config.js            # Configuración bundler
├── styles.css                  # Estilos globales
│
├── src/
│   ├── core/                   # Funcionalidad principal
│   ├── stores/                 # State management (Zustand)
│   ├── types/                  # Definiciones TypeScript
│   ├── components/            # Componentes UI
│   │   ├── dashboard/         # Dashboard de proyectos
│   │   ├── context/           # Documentos de contexto
│   │   ├── files/             # Gestión de archivos
│   │   ├── stages/            # 10 etapas
│   │   ├── calendar/          # Calendario
│   │   ├── routine/           # Rutina diaria
│   │   └── shared/            # Componentes compartidos
│   ├── utils/                 # Utilidades
│   ├── constants/             # Constantes
│   └── hooks/                # Custom hooks
│
├── docs/                      # Documentación técnica
│   ├── API_REFERENCE.md
│   ├── DEVELOPMENT.md
│   └── OBSIDIAN_PLUGIN_GUIDELINES.md
│
└── dist/                      # Archivos compilados
    ├── main.js
    ├── manifest.json
    └── styles.css
```

Para más detalles técnicos, consulta [PLUGIN_STRUCTURE.md](PLUGIN_STRUCTURE.md)
